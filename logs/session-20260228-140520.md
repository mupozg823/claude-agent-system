## 수정된 이슈

### 1. stepTimeout 2분 → 5분
- claude -p가 2분 내 응답 못하는 경우 ETIMEDOUT 발생
- 300_000ms로 증가

### 2. _claudeP spawnSync ENOENT
- spawnSync + shell:false → Windows에서 claude 경로 해석 불가
- spawnSync + shell:true → cmd.exe 사용 → $(cat ...) 미지원
- **해결**: execSync로 복원 (Git Bash 환경)

### 3. decompose 빈 응답 (0 chars)
- 원인: --json-schema + $(cat ...) 조합이 cmd.exe에서 미동작
- spawnSync의 shell:true가 cmd.exe를 사용하여 bash 문법 미지원

### 4. session-init 체크포인트 로딩 간섭
- claude -p가 이전 세션 체크포인트를 로드하여 "이미 완료" 응답
- **해결**: ORCH_DECOMPOSE env 변수로 session-init.js 컨텍스트 로딩 스킵

### 5. --allowedTools '[]' + max-turns 소진
- 도구 없이도 훅이 턴 소비하여 "Reached max turns" 에러
- **근본 해결**: 규칙 기반 분해기(rule-based decomposer) 구현
  - 14개 한국어/영어 패턴 매칭
  - 연결어 분할 (후/하고/그리고/+/&&/→/->)
  - \b 워드 바운더리 제거 (한국어 호환)
  - claude -p는 복잡한 목표만 fallback으로 사용
