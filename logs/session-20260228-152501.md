# 세션 로그: OpenClaw vs 현재 시스템 비교분석

## 날짜
2026-02-28

## 완료된 작업

### OpenClaw 아키텍처 리서치
- GitHub 238K+ 스타, MIT, Peter Steinberger 창작
- 3계층: Gateway(제어평면) → Channel(어댑터) → LLM(Provider)
- 핵심 패턴: Steer(도구 경계 주입), Promise Queue, InboundGuard, Heartbeat
- 5가지 입력 벡터: Message/Heartbeat/Webhook/Hook/Cron
- 50+ 메신저 통합, ClawHub 3000+ 스킬

### 현재 시스템 분석 (4파일 3019 LoC)
- gateway.js (913줄): WS제어평면, InboundGuard, PromiseQueue, AuditTailer
- orchestrator.js (826줄): FSM + 규칙기반 분해(14패턴) + DAG 실행
- relay-supabase.js (757줄): Supabase 양방향 릴레이 + Worker Loop
- agent-engine.js (499줄): Lane Queue + 체크포인트 + 메트릭

### 비교분석 결과

#### 핵심 격차 3가지
1. **Steer 패턴**: OpenClaw는 도구 경계에서 방향 전환 가능, 우리는 큐 레벨만
2. **Worker Loop 지연**: 5초 폴링 vs OpenClaw in-process 즉시 실행
3. **서브스텝 관찰 불가**: 오케스트레이션 중 개별 진행상황 미반영

#### 우리가 더 나은 점
- 카이로소프트 게임화 대시보드 (콤보/XP/업적/날씨/캐릭터)
- 규칙 기반 분해기 14패턴 (API 호출 없이 80% 처리)
- Supabase 릴레이 (모바일 원격 모니터링)
- 영어+한국어 이중 키워드

### 3단계 개선방향 제안
- Phase 1: 대시보드 실시간 동기화 강화 (도구별 애니메이션, 감정, 날씨 연동)
- Phase 2: 아키텍처 격차 해소 (Steer, 즉시 실행, Webhook/Cron)
- Phase 3: 게임성 강화 (스킬트리, 자동 트리거, 협업 애니메이션)
